NBA Player Predictor

PYTHON · FASTAPI · ML · K8S

A backend web application predicting an NBA player's next-game point total against a specific opponent, built as a full production ML system rather than just a notebook.

Trained a RandomForestRegressor on real player/team IDs pulled from PostgreSQL, engineering average-based features (player scoring average, opponent points allowed, team scoring average) so the model could reason about matchups instead of treating IDs as opaque categories.

Tracked 10 experiment runs in MLflow (varying tree count, depth, and test split) and promoted the lowest-MAE run to the MLflow Model Registry under a production alias.

The trained model is served from its own containerized service (separate from the API, so it can scale and redeploy independently), and the whole system ships via a CI/CD pipeline on GitHub Actions - tests and migrations run on every push, QA deploys automatically to a GKE cluster, and production deploys require manual approval through a GitHub Environment gate.

tech stack

Python · FastAPI · PostgreSQL · scikit-learn (RandomForestRegressor) · MLflow (tracking + Model Registry) · Docker · Kubernetes (GKE) · GitHub Actions (CI/CD)
